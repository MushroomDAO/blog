# xiaoheishu.xyz — 产品规划文档

> 域名：xiaoheishu.xyz  
> 定位：人人可用的内容自动发布平台 + GEO 增长服务  
> 日期：2026-04-24

---

## 一、产品愿景

把本 blog 的多平台自动发布能力（M1 Astro/M2 微信/M3 小红书）产品化，让任何内容创作者——尤其是美食博主、旅行博主、KOL——无需技术背景即可一键完成跨平台发布，并自动获得 AI 引擎的 GEO 加持。

**核心命题**：写一次，AI 帮你发布到全平台，并让你的内容被 AI 搜索引擎引用。

---

## 二、用户分析

### 目标用户画像

| 用户类型 | 需求痛点 | 付费意愿 |
|---------|---------|---------|
| 美食/旅行博主 | 每次发布需要在 5+ 平台重复操作，耗时 2-4 小时 | 中高 |
| 内容创作者/KOL | 需要个人品牌域名，图床，稳定发布 | 中 |
| 个人独立博主 | 想拥有自己的站点但不懂技术 | 低-中 |
| 中小企业/本地商家 | 需要持续内容曝光，想被 AI 引擎推荐 | 高 |

### 美食博主时间成本分析

典型美食博主单次发布流程：
- 写稿：60-90 分钟
- 配图处理：30-45 分钟
- 小红书发布（手动排版）：20-30 分钟
- 公众号发布（HTML 格式化）：20-30 分钟
- 个人博客发布：15-30 分钟
- **合计：2.5-3.5 小时/篇**

xiaoheishu 目标：**压缩到 15 分钟**（只需写稿+选图）

## 四、技术架构规划

### 核心技术栈选型

```
前端：         Next.js 15 (App Router) + Tailwind CSS
编辑器：       Tiptap (Markdown + 富文本双模)
后端：         Node.js + Fastify (轻量高性能)
数据库：       PostgreSQL (用户/文章) + Redis (队列/缓存)
文件存储：     Cloudflare R2 (图床，S3 兼容)
博客静态托管：  Cloudflare Pages (每用户独立部署)
发布队列：     BullMQ (Redis-backed，支持重试)
自动化浏览器：  Playwright / Computer Use (平台登录态维护)
```

### 多平台发布技术方案

| 平台 | 方案 | 技术实现 |
|------|------|---------|
| 个人子站 | Astro 静态生成 + CF Pages | 现有 M1 pipeline |
| 微信公众号 | 官方 API（草稿+发布） | 现有 M2 pipeline (Node.js) |
| 小红书 | Playwright 自动化 / MCP 服务 | 现有 M3 + xiaohongshu-mcp |
| 知乎 | Playwright 自动化 | 新建 |
| 抖音/快手图文 | Computer Use（复杂交互场景） | 待研究 |

### Computer Use 探索方向

对于平台登录态维护和复杂交互（如抖音图文发布流程），可尝试：
- Anthropic Computer Use API（截图+点击操作序列）
- 优先用于：无官方 API 的平台，手机端操作（通过 ADB 模拟）
- 风险：稳定性依赖 UI，需要 fallback 机制

### GEO 自动化服务架构

```
用户提交文章
    ↓
LLM 分析提取关键实体（餐厅/菜品/地点/价格）
    ↓
自动生成可引用陈述句（3-5条 blockquote）
    ↓
注入 JSON-LD Schema（BlogPosting/Review/LocalBusiness）
    ↓
生成英文摘要（提升国际 AI 引用率）
    ↓
发布 + 提交 sitemap 到 IndexNow（必应/Yandex 即时收录）
    ↓
每月生成 AI 引用追踪报告
```

## 五、GEO 服务核心价值

### 为什么内容创作者需要 GEO 服务

2026年 AI 搜索用量趋势：
- Perplexity 月活：突破 1 亿（2025）
- ChatGPT Search：集成入默认搜索（2025 Q4）
- Google AI Overview：覆盖 60%+ 搜索结果页

**核心转变**：用户正在从"搜索引擎点击"转向"AI 引擎直接获取答案"。

传统 SEO 让内容出现在搜索结果第一页；GEO 让内容被 AI 直接引用并朗读给用户。

### 美食博主 GEO 服务的特殊价值

```
用户问 AI："成都有哪些好吃的火锅推荐？"
AI 引用："根据 [博主名] 的实测记录，[餐厅名] 的 [菜品] 
         价格约 XX 元，[描述]，适合 [场景]。"
```

**这比任何付费推广都更有公信力**，因为 AI 引用 = 第三方背书。

| 风险 | 概率 | 对策 |
