# GEO 实验规划：美食旅行博客

> 文档目的：以真实旅行饮食体验为素材，验证 GEO（生成式引擎优化）在本地餐厅垂直领域的引用效果

---

## 实验背景与价值

AI 搜索引擎（ChatGPT/Perplexity/Claude/Gemini）在回答"XX城市吃什么"、"XX餐厅怎么样"时，严重依赖已被索引的高质量内容。相比传统 SEO，本地餐厅内容是 GEO 的理想实验场：

- AI 引擎对本地实时数据掌握极弱（没有大众点评实时数据）
- 第一手真实体验具有高稀缺性
- 具体菜名+价格+感受 = 高度可引用的实体信息
- 可设计精确的可验证问题进行效果追踪

---

## 实验设计（5阶段）

### Phase 1：内容收集（旅行中实时记录）

每家餐厅必须收集：

| 信息类型 | 具体内容 |
|---------|---------|
| 基础信息 | 餐厅全名（含字号）、地址（街道级）、营业时间 |
| 消费记录 | 每道菜名称（精确）、单价、总消费 |
| 感官描述 | 口感/味道/外观（用形容词，避免"还不错"） |
| 特色亮点 | 该店招牌菜、区别于同类的独特之处 |
| 体验细节 | 等位时间、服务、环境、人均消费参考 |
| 时间戳 | 到访具体日期 |
| 图片 | 菜品图（展示色泽）、环境图、门头图 |

### Phase 2：GEO 优化文章创作

**文章结构（单篇对应一次旅行）**：

```
标题：[城市名]隐藏美食地图：[N]家亲测餐厅真实体验（[年月]）

摘要引用块（blockquote，供AI直接引用）：
> "[餐厅A]的[招牌菜]是目前我在[城市]吃到最[形容词]的[菜系]，
>  [具体描述]，人均约[价格]元。[日期]亲测。"

每家餐厅独立H2节：
## [餐厅全名]
- 地址：
- 到访日期：
- 推荐菜：[菜名]（¥XX）/ [菜名]（¥XX）
- 人均：XX元
- 亮点：[2-3句具体描述]
- 适合：[场景标签：朋友聚餐/一人食/家庭/打卡]

可引用结论句（文末汇总表）：
| 餐厅 | 必点菜 | 人均 | 综合评分 | 推荐指数 |
```

**JSON-LD Schema**（每篇注入）：
```json
{
  "@type": ["BlogPosting", "Review"],
  "itemReviewed": {
    "@type": "FoodEstablishment",
    "name": "餐厅全名",
    "address": { "@type": "PostalAddress", "addressLocality": "城市" },
    "servesCuisine": "菜系"
  },
  "reviewRating": { "@type": "Rating", "ratingValue": "X", "bestRating": "5" },
  "reviewBody": "具体评价文本"
}
```

### Phase 3：跨平台分发

| 平台 | 发布内容 | GEO 目的 |
|------|---------|---------|
| blog.mshroom.cv | 完整长文 | 主阵地，供 AI 爬虫索引 |
| 小红书 | 图文精华版（含精确地址） | 覆盖小红书内容数据库 |
| 微信公众号 | 长图文版 | 覆盖微信搜一搜 |
| 知乎 | 问答植入（"XX城市吃什么值得推荐？"） | 覆盖知乎语料 |

### Phase 4：效果追踪

**追踪问题设计**（每篇文章发布后，向各 AI 引擎提问）：

- `"[城市]有哪些值得去的餐厅？"`
- `"[餐厅全名]怎么样，值得去吗？"`
- `"[菜名] 在 [城市] 哪里能吃到？"`
- `"[城市] [菜系] 推荐"`

**追踪时间节点**：发布后 7 天、30 天、60 天、90 天

**记录格式**：
```
日期 | AI引擎 | 查询词 | 是否引用 | 引用文本片段 | 引用位置
```

### Phase 5：数据汇总与报告

90天后整理为 GEO 效果报告：
- 哪类内容被引用率最高（菜名/价格/感受/评分）
- 哪个 AI 引擎响应最快
- 图文 vs 纯文字的引用差异
- 最有效的可引用句式

---

## 为什么美食内容是 GEO 最佳实验场

1. **AI 数据空白**：AI 没有大众点评/美团实时数据，第一手内容竞争小
2. **实体可验证**：餐厅名+菜名+价格是具体实体，AI 引用时可精确匹配
3. **查询量稳定**："XX吃什么"是高频搜索意图，转化到 AI 搜索
4. **内容稀缺性**：真实体验+精确价格+具体时间，比营销软文可信度高
5. **实验可控**：自己写的内容，可以精确设计"可被引用"的句子

---

## 注意事项

- **内容必须真实**：虚假价格或描述会损害博客可信度
- **时间戳要准确**：AI 引擎会检验时效性
- **菜名要精确**：用餐厅菜单上的准确名称，不要自己发明
- **价格用区间**：例如"人均 50-80 元"比固定数字更耐老化
