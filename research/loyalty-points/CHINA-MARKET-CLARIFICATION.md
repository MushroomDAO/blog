## §1 中国大陆 loyalty 行业现状（万亿级市场）

### 1.1 五大类别

#### 类别 1：航空里程（你提到的）

| 公司 | 会员数 | 监管 |
|------|--------|------|
| **国航 PhoenixMiles** | **>9,300 万**（2024-11，30 周年） | 民航局 + 单用途预付卡（MOFCOM Order 9）|
| **东航万里行** | 数千万 | 同上 |
| **南航明珠俱乐部** | 数千万 | 同上，SkyTeam 联盟 |
| **海航金鹏** | 数千万 | 同上 |

**为什么航空可以？**
- ✅ **单用途**：积分仅在航司自己 + 集团子公司兑换机票 / 升舱 / 行李
- ✅ **航司本身是受民航局监管的合法实体**，发积分是营销手段
- ✅ **联盟（Star Alliance / SkyTeam）是双边合同协议**，不构成"开放式多商家积分"
- ✅ **没有代币化** — 中央数据库记录里程

#### 类别 2：银行信用卡积分

| 公司 | 规模 | 监管 |
|------|------|------|
| **招行掌上生活** | 数亿信用卡 + 19M 星巴克联名 | 银保监 + PBOC |
| **工行积分** | 数亿用户 | 同上 |
| **建行 e 积分** | 数亿用户 | 同上 |
| **中信积分** | 数千万 | 同上 |

**为什么银行可以？**
- ✅ 银行本身是持牌金融机构，发积分是金融营销
- ✅ 仍然是**单用途**（兑换银行内部服务 / 合作商家商品）
- ✅ 跨行联盟（如银联 U 享 Plan）走 PBOC 牌照通道

#### 类别 3：互联网巨头积分

| 公司 | 积分 | 监管 |
|------|------|------|
| **Alipay 集分宝** | 数亿用户 | Ant 是持牌支付机构（PBOC 牌照） |
| **WeChat Pay 优惠券** | 数亿 | Tenpay 持牌 |
| **JD 京豆** | 数亿 | JD Plus 付费会员体系 |
| **美团积分** | 数亿 | 美团自营闭环 |
| **拼多多多多币** | 数亿 | 拼多多自营闭环 |

**为什么巨头可以？**
- ✅ **超级 App 内部闭环**：积分仅在 App 内消费
- ✅ Alipay / WeChat Pay 本身有 PBOC 支付牌照
- ✅ 内容性质："营销奖励"而非"独立流通价值"

#### 类别 4：第三方 loyalty 平台

| 公司 | 业务 | 监管 |
|------|------|------|
| **积加网络**（杭州） | 跨商家积分流转 | 部分有 PBOC 支付牌照 |
| **积分宝**（北京） | 积分兑换商城 | 走"促销服务"路径 |
| **百望股份** | 含积分相关业务 | 上市公司，全面合规 |
| **客如云** | 餐饮 SaaS 含积分 | SaaS 服务商不接触资金 |
| **微盟会员** | SaaS 会员系统 | 同上 |

**这些公司怎么做的？**

**路径 A**：申请 PBOC 支付牌照（多用途预付卡牌照）
- 资本要求高（亿元级）
- 申请周期 12-24 个月
- 适合大型平台

**路径 B**：走"SaaS / 营销服务"路径
- 自己**不接触用户资金**
- 仅给商家提供工具（数据库 / 短信 / Dashboard）
- 商家自己持有积分发行权
- **不需要支付牌照**
- 适合中小型 SaaS（最多公司走这条路）

#### 类别 5：联盟积分（最接近 Mycelium 想做的）

| 公司 | 业务 | 监管 |
|------|------|------|
| **银联 U 享 Plan** | 银联跨银行积分互兑 | 银联本身是持牌清算机构 |
| **航旅纵横** | 航空 + 酒店 + 旅行 多边联盟 | 持有民航局背景 + 多家航司合作 |
| **京东 PLUS + 爱奇艺** | 平台间会员互通 | 双边合作 |

**关键观察**：联盟积分在大陆**只在持牌实体之间做**（银联、各大行、航司、超级 App），**没有"中小商家自组织联盟"的先例**。

### 1.2 中国大陆 loyalty 市场规模

|------|------|------|

**结论**：中国大陆 loyalty **不是禁区，是一个巨大的合法市场**。我之前的"永远不碰"是严重表述错误。

## §2 那 Mycelium 在中国大陆的真实障碍是什么？

不是"loyalty 业务被禁"，是**Mycelium 的 5 个核心特性**与大陆监管的冲突：

| Mycelium 设计 | 中国大陆监管立场 | 冲突等级 |
|-------------|---------------|---------|
| **1. 区块链发行积分（链上 token）** | [PBOC 2017-09-04 ICO 禁令](http://www.pbc.gov.cn/) + [2021 加密币 10 部门联合通知](https://www.gov.cn/) — "任何形式的代币融资行为禁止" | 🔴 **致命** |
| **2. 跨商家自组织联盟** | 多用途 → 需 [PBOC 支付牌照](https://www.pbc.gov.cn/) — 自组织 = 无主体监管 = 不被接受 | 🔴 **致命** |
| **3. 数据可上链** | [个保法 + 数据安全法](http://www.gov.cn/) — 跨境数据流严格管理 | 🟡 高 |
| **4. 去中心化协议** | 监管要求"**可控可管理**"，去中心化协议无监管对象 | 🔴 **致命** |
| **5. 境外公司直接服务大陆用户** | 跨境支付 / loyalty 服务监管模糊 + VPN 检测 | 🟡 高 |

**三个 🔴 致命冲突**让 Mycelium 当前形态在大陆**不可行**。

**结果**：大陆版可以做，**但本质上等于"中国版积加网络 + AI 优化器"**——已经不是 Mycelium 协议。

## §4 那为什么"早期不做大陆"仍是合理决策？

### 4.1 不是因为"做不了"，是因为"分散精力"

| 路径 | 工程量 | 监管理解 | 销售难度 |
|------|-------|---------|---------|
| Mycelium 香港路径 | 1× | 1× | 1× |
| Mycelium 大陆降级路径 | **2-3×**（完全重写）| **3-5×**（完全不同法规）| **2×**（B2B 销售文化不同）|

**早期 2 个联创的精力**承受不起两条路径并行。

### 4.2 中国大陆 loyalty 市场已经**高度饱和**

不是"蓝海"等你进入：

- 头部：阿里 / 腾讯 / 京东 / 美团 / 拼多多（巨头闭环）
- 第二梯队：积加网络 / 积分宝 / 客如云 / 微盟（SaaS 各占山头）
- 长尾：每个连锁有自己的会员系统（已被 SaaS 服务商占领）
- **新进入者要拼成本 + 关系 + 销售队伍**——大陆 SaaS 销售平均 2-3 年回本

**对比**：清迈 / 香港 / 东南亚的中小商家**仍未被很好地服务**——这是 Mycelium 的真实蓝海。

### 4.3 政策不确定性

| 不确定性 | Mycelium 路径影响 |
|---------|-----------------|

| 个保法跨境数据流细则 | 影响境外服务大陆用户 |

**这些政策窗口可能 2-5 年内变化**——届时再评估更稳妥。

## §4.5 为什么早期不做大陆？真实的 5 个原因（深挖版）

我之前给的 3 个理由（监管冲突 / 市场饱和 / 精力分散）**没有真正抓到核心**。让我重新分析：

### 真实原因 1：监管"未明确允许"的灰色地带 = 创业死亡区

**大陆监管的根本特性**：**不是明确禁止特定事，而是默认"未明确允许的事都不允许"**。

具体到 Mycelium：

| 业务行为 | 大陆是否有明确法律框架？ |
|---------|----------------------|
| 商家发行积分（单用途）| ✅ MOFCOM Order 9 明确允许 |
| 跨商家积分流通（多用途）| ⚠️ 必须 PBOC 支付牌照 |
| 区块链发行积分 token | ❌ **没有任何法律框架** — 不是禁止，是**没有许可路径** |
| 用户跨境使用境外积分协议 | ❌ **没有任何法律框架** |
| 自组织联盟 / DAO 结算 | ❌ **没有任何法律框架** |

**没有法律框架** = 监管随时可以认定你违规，而你**没有合规路径可走**。

这对早期初创**是致命的**。大陆监管特点：**今天合法的事情明天可能被宣布违规**（2021 教育双减、互联网反垄断、加密币禁令都是突然），追溯执行常见。**早期 2 联创承受不起这个风险**。

### 真实原因 2：哲学根本冲突 — "无许可" vs "持牌经济"

这是最核心的冲突，我之前没指出：

| Mycelium 设计哲学 | 中国大陆监管哲学 |
|-----------------|----------------|
| 任何商家**无需许可**加入协议 | 任何金融 / 支付 / 类货币业务**必须持牌** |
| 协议**自组织**形成联盟 | 任何联盟 / 共享业务**必须有可问责主体** |
| 智能合约**自动执行规则** | 监管要求"**人为可控**" |
| **去中心化**协议本身 | 监管要求"**可监管对象**" |

**关键**：即使 Mycelium 选择**降级**进大陆（去掉链上、去 Mycelium 协议名义、申请支付牌照），它就**不是 Mycelium 了**——这就是另一家做积分 SaaS 的中国公司，与积加网络竞争。

**所以不是"做不了"，是"做了就不是 Mycelium"**。

### 真实原因 3：数据合规成本（远超你想象）

如果境外 Mycelium 公司服务大陆用户，要满足三法合一：

- [《个人信息保护法》](https://www.gov.cn/xinwen/2021-08/20/content_5632486.htm)（2021-11-01 生效）
- [《数据安全法》](https://www.gov.cn/zhengce/2021-06/11/content_5616919.htm)（2021-09-01 生效）
- [《网络安全法》](https://www.gov.cn/xinwen/2016-11/07/content_5129723.htm)（2017-06-01 生效）

具体义务：

| 要求 | 成本 |
|------|------|
| 在大陆设立机构（个保法 §53）| HKD 100K+ 一次性 |
| 数据本地化存储（个人信息）| 服务器 + 运维 HKD 50K+/年 |
| 跨境数据传输安全评估（CAC 审批）| 每次评估 6-12 个月，成功率不确定 |
| 重大数据出境审批 | 同上 |
| 用户单独同意机制 | 法律文书 + UI 改造 HKD 50K+ |
| 个人信息保护影响评估（PIA）| 每年 HKD 30K+ |

**总计**：在大陆合规运营 ≈ HKD 300K+/年合规成本（不含罚款风险）。

**对比**：香港 Schedule 8 路径年合规成本 ≈ HKD 50K/年。

**6 倍合规成本差异**，对早期创业是负担。

### 真实原因 4：中国大陆 loyalty 已有"事实标准玩法"

大陆中小商家的真实习惯：

| 大陆现状 | Mycelium 难以替代 |
|---------|-----------------|
| 微信小程序 + 微信支付内嵌积分 | 用户已习惯，不会切换 |
| 美团 / 大众点评 / 抖音生活服务自带会员系统 | 平台流量绑定，商家依赖 |
| 收银机 SaaS（客如云 / 银豹 / 微商城）含积分 | 已嵌入商家工作流 |
| 微信群 + 朋友圈 + 私域流量 | 营销已"社交化"，不需要协议层 |

**Mycelium 的卖点**（无许可 / 联盟 / 区块链）在大陆**没有真实需求**：
- 商家不在乎"无许可" — 他们已经在用美团 / 抖音
- 商家不在乎"联盟" — 微信群和朋友圈已经互通客户
- 商家不在乎"区块链" — 用户更不在乎

**对比清迈 / 东南亚**：没有美团 / 抖音的等价物，碎片化 SaaS 市场，Mycelium 反而是空白填补。

### 真实原因 5：早期资源有限的战略选择

**这条单独不构成"不做大陆"的理由，但配合上面 4 条 = 决策**：

- 2 个联创 + 有限资金 + 18 个月跑道
- 选 1 个市场打透 vs 分散在 2 个截然不同的市场
- **机会成本**：在大陆耗时 18 个月才能合规，**同样时间在东南亚可以拿下 50 家商家**

### 总结：不是禁区，是"对早期初创的反 ROI 市场"

| 维度 | 大陆 | 东南亚 / 香港 |
|------|------|-------------|
| 法律框架 | ❌ 无对应 | ✅ Schedule 8 / Limited Purpose |
| 合规成本 | HKD 300K+/年 | HKD 50K/年 |
| 监管风险 | 不可预测 | 可预测 |
| 市场密度 | 已被巨头覆盖 | 碎片化空白 |
| 哲学契合度 | "持牌"vs"无许可"冲突 | 友好 |
| 创业时间 ROI | 18+ 月才能起步 | 3-6 个月起步 |

**结论**：早期不做大陆**不是因为做不了，是因为对早期初创的 ROI 太差**。

**3-5 年后值得重新评估**：
- 如果 Mycelium 在东南亚 / 香港验证成功 → 资源更多 → 可以承担大陆合规成本
- 如果中国大陆放开区块链 loyalty / RWA 应用 → 法律框架可能出现
- 如果 Mycelium 与积加网络等持牌公司合作（白标）→ 借助他们的合规身份
