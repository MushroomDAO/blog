# Mycelium 积分协议合规设计建议

> 起草：2026-04-28 · 基于 [REPORT.md](./REPORT.md) 的事实基础
> 类型：可执行决策建议
> 关键问题：**如何既合法合规，又利用区块链特性？**

---

## 一句话建议

> **把 Mycelium 积分协议定位成"给中小商家的开放协议对账基础设施 + AI 发行优化器"，香港注册，Cap. 584 Schedule 8 豁免起步，把所有合规规则嵌入到智能合约里（而非 ToS）；不要做 token 项目（即使技术上是 token），不要 AMM 上市，不要做用户行为预测；用商家 SaaS 订阅 + 联盟匹配费做收入；3 年内不碰中美监管深水区。**

---

## 8 个具体决策

### 决策 1：起步司法管辖 = **香港**

| 选项 | 法律风险 | 启动成本 | 推荐度 |
|------|---------|---------|-------|
| **🟢 香港** | 低（Schedule 8(3) 直接豁免不可换现的 loyalty SVF）| 低（一周注册）| **首选** |
| 🟢 新加坡 | 低（MAS Limited Purpose 例外友好） | 中 | 备选 |
| 🟡 UAE DIFC | 低（创新执照，Loyyal 走过这条路） | 中 | 第三 |
| 🟡 美国 Wyoming | 中（CFPB UDAAP + 各州 escheat 零碎） | 中 | 不优先 |
| 🟡 日本 | 中（PSA 第三者型注册 + 50% 担保金） | 高 | 不优先 |
| 🔴 韩国 | 高（EFTA 2024 100% 备付金） | 极高 | 不做 |
| 🔴 **中国大陆** | **极高**（2017 ICO 禁令 + 2021 加密币禁令）| 不可行 | **绝对不起步** |

**为什么是香港不是新加坡**：
- HK Cap. 584 Schedule 8(3) 对忠诚度 SVF 给出明确豁免（不可换现前提下）
- 香港稳定币条例（2025-08-01）恰好画清了"loyalty 不是 stablecoin"这条线
- 区块链技术中性 — Schedule 8 豁免不区分底层技术
- 与 mushroom.cv 域名 + AAStar 生态时区 / 法律体系兼容
- 必要时可升级到 SVF 牌照（HKD 25M 资本，未来再说）

### 决策 2：积分定义 = **service voucher，不是 currency / token**

这是最重要的一条。把这个定义**嵌入到智能合约里**，不靠用户协议（ToS）。

协议级强制规则：

```
1. 仅持牌商家可发行积分（OpenPNTs ERC-1155 合约白名单）
2. 每发行 1 积分必须对应商家承诺的真实 SKU 可兑换义务
3. 积分代币默认 soulbound（兑换前不可转让）
4. 兑换仅可兑商家或联盟成员的商品/服务，不可换法币
5. 每个商家的积分必须有声明的有效期（建议 2-3 年），过期链上自动作废
6. 协议级禁止在 AMM/DEX 上市；联盟内白名单转移可（带额度）
```

**这一条解决了**：
- ✅ FinCEN closed-loop 豁免（不可换现）
- ✅ SEC Howey 排除（"通过消费获得"，不是投资品）
- ✅ MiCA Art. 4(3)(c)/(d) 例外（utility token + limited network）
- ✅ HK Cap. 584 Schedule 8(3)（非 cash-redeemable bonus point SVF）
- ✅ 中国 2017 ICO 禁令（不可换法币、不跨第三方支付）
- ✅ EU EMD2（不构成 e-money，因不是支付工具）

### 决策 3：联盟设计 = **Flash Coalition Smart Contract**

联盟形成必须满足 4 个条件（智能合约执行）：

```
1. 联盟必须有合同协议（MOA on-chain）— 满足"contractually affiliated group"
2. 联盟必须有终止时间或终止条件（保持"临时性"）
3. 联盟内积分仅在白名单成员间可转移
4. 跨联盟清算用 ZK 证明（不暴露用户身份）
```

**为什么"临时" + "合同"是合规的关键**：
- PSD2 Art. 3(k)(ii) Limited Network Exclusion 要求"有限网络服务提供方有直接商业协议"
- FinCEN closed-loop 要求"在指定商家或地点使用"
- HK Schedule 8 要求"有限用途"

**临时性**避免了 Plenti 长期联盟的"放牧猫"问题；**合同性**满足全球 LNE 类条款的核心要件。

### 决策 4：硬上限（避开 MSB 触发）

```
每用户每商家每日发行/兑换 ≤ 100 USD 等价     # 远低于 FinCEN $2,000/day
每联盟每月总流量 ≤ 80,000 EUR 等价           # 远低于 EU EUR 1M LNE 通报阈值
单商家年发行量 ≤ 1,000,000 USD 等价（默认）   # 满足 HK / 新加坡小规模豁免
```

商家如果要突破这些上限 → **必须主动升级到持牌通道**（Mycelium 帮助申请 HK SVF / 新加坡 MPI / 日本第三者型）。

### 决策 5：AI 角色 = **商家发行率优化器**，不是用户行为预测

明确 AI 用什么、不用什么：

✅ **该用 AI**：
- 帮商家找最优积分/美元比（"我是面包店，发多少分合理？"）
- 联盟匹配（咖啡店 + 健身房客户互补度评分）
- 欺诈检测（账户异常 + 商家自伤异常）
- 跨联盟自动清算（Loyyal-Emirates 80% 对账节省的复制）

❌ **不该用 AI**：
- 用户行为预测 / 个性化定价 → **GDPR Art. 22 自动化决策风险**
- 用户画像营销 → **GDPR + 中国个保法 + EDPB Guidelines 1/2024 都需要明确同意**
- 信用评分 → **触发金融监管**

**核心差异化**：把 Albertsons / Starbucks 内部团队级的优化能力开放给 200 客户的小面包店。这不是个性化（个性化是入门券），这是中小商家从未拥有过的能力。

### 决策 6：区块链特性的取舍

✅ **该用的**：
- 智能合约固化兑换规则（防 CFPB UDAAP 类贬值风险，这是合规护城河，不是技术炫耀）
- ERC-1155 多商家账本（gas 高效，每个 tokenId = 一商家）
- ERC-4337 paymaster — **让用户用积分本身付 gas**（关闭循环，移除 ETH 依赖）
- ZK 证明（兑换资格验证 + 无 PII 上链）
- Merkle 证明批量发放（避免链上膨胀）
- Soulbound（兑换前不可转让）

❌ **不该用的**：
- 二级市场 / DEX 上市（kills 所有合规豁免）
- 跨链桥（增加攻击面，对临时联盟没价值）
- DAO 治理积分本身（让积分变成证券）
- Stablecoin 风格 1:1 法币锚定（触发 GENIUS Act + MiCA EMT）
- 可升级合约（合约一旦可任意修改，CFPB UDAAP 风险回来）

### 决策 7：商业模型 = **保险 + SaaS**，不靠积分本身

❌ **不能靠**：
- 积分本身收交易费（污染服务凭证定性，让你看起来像支付机构）
- 投机性发行预购积分（kills 无对价豁免）
- 积分上 AMM 后的流动性挖矿手续费（kills SEC + MiCA + 中国 ICO 禁令豁免）

✅ **应该靠**：
- **商家订阅费（SaaS）**：基础协议免费 + AI 优化器 + 商家分析 dashboard 收费
  - 参考：Currency Alliance 收 ~2% 兑换值；我们走 fixed monthly fee 更友好
  - 价位：HKD 200-2000/月（按规模）
- **联盟匹配服务费**：一次性帮商家找互补伙伴（HKD 500-5000/匹配）
- **企业升级版**：白标 + 私有化部署中型商家（HKD 50,000+/年）
- **可选：保险池**：商家可选购"积分负债保险"，由协议精算分摊（这是合规附加值）

参考 Loyyal 的存活路径：去 token，卖企业 SaaS。

### 决策 8：起步路径

**Phase 1（M1-M3）**：

## 6 道绝不能跨的红线

1. ❌ **永远不允许法币购买积分**（kills 所有 PSA / EFTA / EMD2 豁免）
2. ❌ **永远不允许积分换法币**（kills FinCEN closed-loop / HK Schedule 8 豁免）
3. ❌ **永远不在 AMM / DEX 上市**（kills SEC / MiCA / 中国 ICO 禁令豁免）
4. ❌ **永远不接受中国大陆用户早期参与**（直到中国监管立场清晰）
5. ❌ **永远不让积分跨非协议联盟成员转移**（kills "limited network" 定性）
6. ❌ **永远不存储用户 PII 上链**（GDPR + 中国个保法 + 不可逆带来的法律风险）

## 区块链特性的"合规价值"叙事（对外宣传用）

不要说"我们做 Web3 loyalty"。说：

> **"我们提供给中小商家的开放协议积分基础设施，把全球积分行业 30 年的合规最佳实践（FinCEN closed-loop / PSD2 LNE / HK Schedule 8）固化到智能合约里，让你不需要昂贵律师团队就能拥有合规的积分系统。AI 帮你找到最优发行率，区块链让兑换规则不可被单方面贬值（解决 CFPB 2024-12 提出的核心痛点），临时联盟让你和互补商家做活动而不需要长期承诺。"**

这个叙事的好处：
- 强调"合规护城河"而不是"去中心化"（吸取 Plenti / Loyyal / Qiibee 的教训）
- 把 CFPB Circular 2024-07 当作产品差异化（合约不可贬值 = 解决 UDAAP 风险）
- 中小商家听得懂（"不需要昂贵律师"是真痛点）
- 不引发监管 / 银行的敌意（不是要替代银行）

## 关键不确定性（你需要在 Phase 1 验证）

1. **HK Schedule 8 律师意见会不会通过**：Mycelium 的"临时联盟 + 跨商家可转移"是否仍在豁免内？这一条决定整个起步方案是否可行。
2. **种子商家愿意付 SaaS 月费吗**：HKD 200-2000/月 对小商家是否过门槛？需要 PoC 期免费验证。
3. **AI 发行率优化器对小数据是否有效**：Albertsons 的 AI 在百万用户基础上工作；200 客户的面包店数据稀疏，效果可能打折。
4. **联盟需求是否真实存在**：临时联盟听起来美好，但实际商家会不会主动找互补伙伴？还是需要 Mycelium 主动撮合？

## 给创始人的一句话提醒

**你不是在做"区块链 loyalty 项目"，你是在做"把全球积分行业 30 年合规知识平民化的 SaaS"，区块链只是其中最方便的执行工具**。这个心智差异决定了对外叙事、监管姿态、产品优先级、收入模型 — 全都不一样。

参考 Loyyal 改卖 SaaS 才存活的教训，参考 Currency Alliance 一直坚持 SaaS 的路径。**Web3 loyalty 项目 95% 失败率的真正原因不是技术，是定位错了**。
