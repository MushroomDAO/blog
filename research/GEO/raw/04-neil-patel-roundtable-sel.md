## 三、Steve Toth / SEO Notebook & Notebook Agency

### 3.1 **Chunking 实操细节（最具体规则）**

**核心规则**（行业最具体）：
- **Token 长度**：每段落 **100-300 tokens**（约 75-225 英文单词，或 50-150 中文字）
- **结构**：每段必须是 declarative passage（陈述式），不能是过渡或铺陈
- **可独立性测试**：每段必须 self-contained — 不依赖上下文也能被 LLM 抽取理解
- **段落功能**："让 LLM 迅速理解段落含义并将其与查询关联起来，支撑推理路径"

### 3.2 Schema 实操（相对模糊）
- 强调 FAQ schema 改善 chunk-level 抽取
- JSON-LD 是所有主要 AI 引擎依赖的标准
- 6 类核心 JSON-LD：Organization、Article、FAQPage、HowTo、Product、BreadcrumbList
- Organization schema 包含：name、URL、logo、founding date、sameAs links

### 3.3 "AEO is Eating SEO" 关键数据
- 他的代理 **60/40 分布**：60% AEO + 40% SEO（客户需求驱动）
- **MQL 质量**：来自 LLM 的客户比传统搜索 **4-5 倍更合格**
- Surfer CMO 援引：约 20% 的付费客户来自 LLM
- Bing 数据保留：Google 因 privacy 过滤的 30-40% 查询在 Bing Webmaster Tools 仍可见
- Angi 案例：长尾内容策略带来 **79% 转化提升**

### 3.4 转化率论点
- "LLM 买家更 bottom funnel"
- **"Up to 22x higher conversion per visit"** 来自 LLM 流量
- "You're able to qualify the solution that you want way better with an LLM than you ever were on Google."

### 3.5 反 fan-out 立场
**明确反对**为每个可能的 fan-out 查询创建单独页面（你无法预测所有合成子查询）。替代：**按比较、定价、用例、ICP/公司规模分段**。

## 四、**Mintcopy / Norg.ai BLUF +67% 数据 — 已被证伪！**

### 4.1 **关键调研结论：**

> **"BLUF 提升 +67% AI 引用率" 找不到原始研究，极可能是营销话术二次传播的产物**

### 4.2 真实情况拆解

**Mintcopy 原文**（"BLUF: The Ski Ramp Content Strategy"）：
- **没有任何具体百分比**
- 援引 Kevin Indig 但未提供链接或数字
- 完全是定性论述

**Norg.ai 原文**：
- 援引一个 **"Growth Marshal peer-reviewed study, Feb 2026, n=730"** 的研究
- **这个研究在所有公开数据库（Google Scholar、PubMed、arXiv）中找不到任何痕迹**，强烈暗示伪造或包装话术
- 引用其他几个数字均**无来源**

### 4.3 67% 的真实来源（混淆了三个不同概念）：

1. **Kevin Indig Growth Memo**：分析 ~98K ChatGPT 引用 + 12亿响应，**真实 67% 含义**：top 30 domains 占据 67% 的话题级引用 / **完全跟 BLUF 无关** — 这是引用集中度数据
2. **单一 B2B SaaS 案例**："BLUF 实施后 6 个月 organic traffic +67%" — 是流量增长，不是 AI 引用率，且是 n=1 轶事
3. **Norg.ai 引用的不可追溯研究**

### 4.4 真正可信的 BLUF/chunking 数据

- **Aggarwal et al. (ACM SIGKDD 2024)**：GEO 优化 visibility 提升至多 **40%**（唯一可信学界数字）
- **Kevin Indig**：**43.2% top-1 Google 排名页面被 ChatGPT 引用**（vs <13% beyond top-20，**3.5×**）
- 85% 检索的页面从未被引用
- 超过 20K 字符的页面平均 10.18 引用 vs 短页面 2.39 引用

**主报告应剔除"BLUF 提升 67% AI 引用率"说法，改用 Aggarwal 40% 或 Indig 3.5x 数据。**

## 七、**Agentic Commerce Protocol & UCP（GEO 下半场最大变量）**

### 7.1 OpenAI Agentic Commerce Protocol (ACP)

**已公开细节**：
- **维护方**：OpenAI + Stripe，Apache 2.0 开源
- **版本演进**（date-versioning）：
  - 2025-09-29：初始发布
  - 2025-12-12：履约增强
  - 2026-01-16：能力协商
  - 2026-01-30：扩展/折扣/支付处理器
  - **2026-04-17：当前稳定版**（cart、feed、orders、authentication、MCP binding）

**双 API 架构**：
- Checkout API (`openapi.agentic_checkout.yaml`)：发起购买
- Delegate Payment API (`openapi.delegate_payment.yaml`)：安全 token 交换

**关键特征**：商家不直接接触买家支付凭证—支付 token 通过 agent 中介流动；商家保持现有商务基础设施 + 新增 AI agent 发现层。

### 7.2 Google Universal Commerce Protocol (UCP)

**已公开细节**：
- **联合开发者**：Shopify、Etsy、Wayfair、Target、Walmart + 20+ 生态伙伴
- **接入面**：Google AI Mode 和 Gemini，从 direct buying 起步
- **互操作**：与 AP2（Agent Payments Protocol）、A2A（Agent2Agent）、MCP 兼容
- **绑定**：REST API 和 MCP binding 双支持
- **商家保留**：Merchant of Record 状态 + 客户关系/数据/售后所有权
- **入口**：Google Merchant Center 现有 shopping feeds + waitlist

### 7.3 **对内容创作者 / SMB 的实际影响（重大）**

1. **"流量"概念被重定义**：不再是访问网站，而是 "agent 调取你的产品 feed 完成交易"
2. **传统 SEO/GEO 的 ROI 模型部分失效**：用户根本不再点击链接进入网站，而是 ChatGPT/Gemini 直接代为下单
3. **Schema/Feed 投资优先级上升**：JSON-LD Product schema 不仅仅是 SEO 信号，而是 **agent 可执行的购买数据源**
4. **品牌站结构性放大效应**：UCP 把 AI 流量直接钩在 Merchant Center，已有商品库/品牌的商家受益最大；纯内容创作者被边缘化（除非走联盟/评测路径）
5. **小博客/内容站的危机**：传统"AI 引用 → 网站访问 → 转化"漏斗被压缩为"AI 推荐 → 直接交易"——内容站若不接入 ACP/UCP feed，可能完全脱离交易闭环

### 7.4 SMB 实际行动建议（基于公开信息推断）

- 必须接入 Merchant Center / Stripe + ACP，否则错失 Q4 2026 起的 agent 直接购买流量
- 内容站考虑 affiliate 升级：从传统 affiliate link 升级到 ACP-aware product feed
- 监控自家品牌在 ChatGPT/Gemini 的 product carousel 出现频率（成为新的 KPI）
