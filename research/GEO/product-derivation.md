## Step 1：定义不可改变的约束

```
AI 回答 100%
├─ 训练数据   25%   ← 锁死，下次重训才反映（6-12 月延迟）
├─ 实时检索   70%   ← 唯一可影响的填区
└─ 用户上下文 5%    ← 用户粘贴，不归我们管
```

**结论 1**：我们能影响的全部边界 = **70% 实时检索**。

任何宣称"让你影响 AI 训练数据"的产品都是骗子。Lily Ray 已点名批评这种 grift（["Coordinated GEO disinformation campaigns"](https://lilyraynyc.substack.com/p/a-reflection-on-seo-and-ai-search)）。

---

## Step 2：拆解 70% 实时检索的内部结构

```
实时检索 70%（=100%）
├─ Google/Bing Web 搜索    60%    ← 谁排名高，AI 就抓谁
├─ AI 自有爬虫直抓         25%    ← OAI/Perplexity/Claude 的爬虫
└─ 特殊源（Reddit/Wiki/UGC）15%    ← 与传统 SEO 不重合的渠道
```

**数据支撑**：
- [Ahrefs 数据](https://ahrefs.com/blog/ai-traffic-research/)：**76.10% AI Overview 引用页在 Google top 10**——证明 Web 搜索是主要渠道
- 但 **ChatGPT 28% top-cited 在 Google 零曝光**——证明 AI 自有爬虫和特殊源是真实独立通道
- [Lily Ray](https://lilyraynyc.substack.com/p/a-reflection-on-seo-and-ai-search)："Every single URL surfaced in an LLM response is not generated by the model's brain but is pulled from a live search index"

## Step 3：客观量化"什么真正预测引用"

来源：[Ahrefs 75K 品牌相关性研究](https://ahrefs.com/blog/ai-brand-visibility-correlations/)（业内最大样本实证）+ [Kevin Indig 1.2M ChatGPT 响应](https://www.growth-memo.com/p/the-science-of-how-ai-pays-attention)

| 变量 | 相关系数 / 效应 | 含义 |
|------|----------------|------|
| **YouTube 提及** | **0.737** | 最强信号（反共识，多数工具忽略） |
| **品牌网络提及** | **0.664** | 第二强 |
| **品牌锚文本** | 0.527 | 强 |
| **品牌搜索量** | 0.392 | 强 |
| **Google 排名 #1** | **3.5x 效应** | 引用率 43.2% vs rank 20+ 的 12% (Indig 1.2M 实证) |
| **Domain Rating** | 0.326 | 中 |
| **反向链接数** | 0.218 | 弱（被淘汰的旧 SEO 信号） |
| **内容数量** | **0.194** | **基本无关** |

**结论 2**：5 个真正起作用的可控杠杆 = **YouTube 提及 + 品牌网络提及 + 品牌锚文本 + Google 排名 + Chunk 质量**。

**反共识发现**（必须告诉客户）：
- ❌ "内容农场策略"（疯狂发文章）已死——内容数量与可见度几乎无关
- ❌ "建反链"已弱化——0.218 已不是核心信号
- ✅ YouTube 是最强信号——多数工具不监测

## Step 4：把"杠杆"映射到 AI 检索 5 阶段管线

来源：[Mike King iPullRank "How AI Mode Works"](https://ipullrank.com/how-ai-mode-works)

```
用户 prompt
    ↓
[Stage 1] Expansion       —— 拆成 8-28 子查询（Mike King: avg 10.7）
    ↓
[Stage 2] Routing          —— 子查询路由到内容形式（文本/视频/UGC）
    ↓
[Stage 3] Retrieval        —— 从 Web/Crawl/Special 拉候选页面
    ↓
[Stage 4] Selection        —— Pairwise LLM 选 chunk
    ↓
[Stage 5] Synthesis        —— LLM 合成答案（含引用 / 含品牌名）
    ↓
最终 AI 回答
```

**每个阶段我们能不能干预？**

| 阶段 | 可干预？ | 干预方式 | 业界工具是否做 |
|------|--------|---------|--------------|
| Stage 1 Expansion | ❌ 不能 | 只能侦察（Mike King 的 Qforia 模拟器） | iPullRank Qforia 唯一 |
| Stage 2 Routing | ✅ 可以 | 把内容做成对的形式（视频→YT；UGC→Reddit） | 几乎无 |
| Stage 3 Retrieval | ✅ 强可干预 | robots.txt + SSR + Google 排名 | SEO 工具部分覆盖 |
| Stage 4 Selection | ✅ 强可干预 | Chunk 化、BLUF、entity density | **Mike King 直接说"业界没有"** |
| Stage 5 Synthesis | ✅ 部分可干预 | 品牌网络提及决定"被引用 vs 被点名" | 监测有，干预无 |

**关键缺口**：Stage 4 Selection 是 Mike King 公开点名的产品空白：
> "There is no SEO software that will get you passage-level embeddings...you'd have to write your own code to do what I just walked you through."

## Step 5：从可干预阶段推导出 6 Layer 产品架构

按饼图份额排序：

| Layer | 名称 | 影响阶段 | 饼图份额 | 业界覆盖度 |
|-------|------|---------|---------|----------|
| L0 | Monitoring（测量基线） | 全部 | 触发其他所有 Layer | 已有红海（Otterly/Profound） |
| L1 | Discoverability（爬虫可达） | Stage 3 | 17% | 有但分散（robots 检查工具） |
| L2 | Retrievability（Web 排名） | Stage 3 | **42%** | SEO 工具占据 |
| L3 | Citability（chunk 引用） | Stage 4 | 20-25% | **业界空白** |
| L4 | Mention-worthiness（品牌） | Stage 5 + 间接全部 | 30% | 监测有，干预无 |
| L5 | Defense（反 spam） | 全部 | 保护已有 70% | 几乎无 |
| L6 | China Platform Sourcing | Stage 3 + 5（中文） | 中文市场 35% | 完全空白 |

## Step 6：每个 Layer 的完整推导

每个 Layer 说清楚 5 件事：业务背景 / 价值 / 饼图份额 / 技术原理 / 怎么做。

### Layer 0：Monitoring（测量基线）

#### 业务背景
- [Aleyda SEOFOMO 2025 调研](https://hub.seofomo.co/surveys/state-ai-search-optimization/)：**91% 客户问过 AI 可见度**，但 62% 站点贡献 0-5% 收入——张力是"想知道但不知道"
- [Wheelhouse 数据](https://www.wheelhousedmg.com/insights/articles/ai-traffic-is-already-in-your-analytics/)：**70.6% AI 流量被 GA4 错分为 "Direct"**——用户根本没意识到自己已经有 AI 流量
- [Cyrus Shepard](https://legaltalknetwork.com/podcasts/lawyerist-podcast/2025/12/) 反共识："For most people—just tracking your Google visibility is probably going to get you 80% of the way there"——但他承认"对预算大的公司，建立基础 AI visibility 追踪 is reasonable"

#### 价值
- 把"看不见"变成"看得见"
- **触发其他所有 Layer 的动作**——measurement enables everything else
- 反 GEO grift 的诚实定位锚点

#### 饼图份额
0%（不直接改变 AI 输出）+ 杠杆放大器

#### 技术原理
1. **Cron Worker** 每天定时触发（Cloudflare Workers Cron Triggers，免费）
2. **Browser Rendering** 模拟真实用户访问 ChatGPT / Perplexity / Kimi / DeepSeek / 豆包 / AIO，提交配置好的 N 个 prompt（$0.09/小时，10 小时/月免费）
3. **Workers AI** 解析 SERP，提取引用 URL + 品牌名出现位置
4. **Vectorize**（5M 向量免费）存历史快照，对比变化
5. **Ghost Citation 检测**（[Indig 概念](https://www.growth-memo.com/p/the-ghost-citation-problem)）：区分"链接被引用" vs "品牌名被提到"——多数工具不区分

#### 怎么做（产品功能）
- **每日扫描**：N 个 prompt × M 个引擎，存到 D1
- **每周报告**邮件："本周 ChatGPT 引用你 8 次（上周 3）；Gemini 提及你品牌 12 次但只引用 2 次（典型 Ghost Citation）"
- **GA4 反向归因脚本**：把 70.6% Direct 重新拆出 AI 部分
- **变化告警**：引用率突跌时立即邮件（防止 Lily Ray 案例 -49% 没人发现）

### Layer 1：Discoverability（爬虫可达性）

#### 业务背景
- [Aleyda Solis](https://www.aleydasolis.com/en/ai-search/ai-search-optimization-checklist/) 反复强调 ChatGPT/Claude **不执行 JavaScript**——重 JS 渲染网站永远抓不到
- Ahrefs 数据：**GPTBot 被屏蔽率 5.89%**（最高）；**ClaudeBot 屏蔽率 YoY +32.67%**——很多人用 Cloudflare 默认 WAF 误挡了 AI 爬虫
- 多数老站不知道有 OAI-SearchBot / PerplexityBot / Claude-SearchBot 三个**与 GPTBot 不同**的 user-agent

#### 价值
**底线保护**——没这一层，后面什么都不用做（爬不到就一切归零）

#### 饼图份额
**70% × 25% = 17%**（直接爬虫渠道）+ 部分影响 60% Web 搜索（Bing/Google 也需可爬）

#### 技术原理
1. **Workers + Browser Rendering** 用 5 个不同 user-agent（OAI-SearchBot、PerplexityBot、Claude-SearchBot、GPTBot、Bingbot）抓你的网站
2. **检测维度**：
   - HTTP 状态码（403/429 表示被 WAF 拦了）
   - SSR 还是 CSR（用 fetch 拿 initial HTML 看关键内容是否在）
   - robots.txt 是否 Allow（Aleyda 推荐的 5 个 user-agent）
   - Cloudflare Challenge / WAF 是否拦截
3. **Workers AI** 解析 Cloudflare 配置，找出"挡了 AI 爬虫但你不知道"的规则
4. **自动生成正确的 robots.txt + Cloudflare WAF 白名单**（Cloudflare API 可写）

#### 怎么做（产品功能）
- **一键审计**：30 秒返回报告"你的 robots.txt 屏蔽了 OAI-SearchBot——这意味着 ChatGPT 永远不会引用你"
- **可执行的修复**：生成 robots.txt + WAF 规则**直接推送到用户 Cloudflare 账号**（这是 Cloudflare-Native 产品的独特价值）
- **持续监控**：每周扫一次，规则变化告警

### Layer 2：Retrievability（Web 搜索排名）

#### 业务背景
- [Kevin Indig 1.2M ChatGPT 实证](https://www.growth-memo.com/p/the-science-of-how-ai-pays-attention)：**Google rank #1 → ChatGPT 引用率 43.2%；rank 20+ → 12%**（**3.5x 差距**）
- [Lily Ray](https://lilyraynyc.substack.com/p/a-reflection-on-seo-and-ai-search)："Every single URL surfaced in an LLM response is not generated by the model's brain but is pulled from a live search index"——传统 SEO 是 GEO 的根
- [Brian Dean](https://backlinko.com/llm-visibility) 引用 Semrush："Almost 90% of ChatGPT's citations come from search results ranking in positions 21+"——但同时承认 ChatGPT 不完全跟随 Google 排名

#### 价值
**最大杠杆点**——影响最大的可控变量。**已有 SEO 工具的用户用我们 = 让 SEO 投资在 AI 时代继续 ROI**。

#### 饼图份额
**70% × 60% = 42%**——这是最大单一 Layer。

#### 技术原理
1. 接入用户 **Google Search Console API**（OAuth 一次授权）
2. 算法找"**甜区页面**"：
   - 当前排名 4-15（rank up 1 位 → AI 引用率翻倍的高 ROI 区间）
   - 有引用价值（数据型 / 指南型 / 对比型——基于 [Ahrefs 内容类型 AI 流量数据](https://ahrefs.com/blog/geo-generative-engine-optimization/)：Best 7.06%、How-to 6.35%、Vs 4.88%）
   - Click 率低于平均
3. **Workers AI** 给每个甜区页面输出"什么改了能 rank up"建议（基于 SEO 基本功 + Indig 的 1.2M 研究语言学特征）

#### 怎么做（产品功能）
- **仪表盘**："你有 12 个页面在 rank 4-10 区间。把它们推到 top 3，AI 引用率预计翻倍。"
- **优先级排序**：影响 × 容易度（Cyrus Shepard 的 RICE 思路）
- **改进建议**：基于具体页面给"加 X、改 Y、删 Z"

### Layer 3：Citability（chunk-level 可引用性）

#### 业务背景
**这是 Mike King 直接说"业界没有工具"的领域**：
> "There is no SEO software that will get you passage-level embeddings...you'd have to write your own code." ([iPullRank](https://ipullrank.com/how-ai-mode-works))

[Indig 1.2M 实证](https://www.growth-memo.com/p/the-science-of-how-ai-pays-attention) 给出的具体写作参数：
- **44.2% 引用来自页面前 30%**（Ski Ramp Pattern）
- **Definitive language**（"X is Y"句式）→ +14% 引用率
- **Entity density 20.6%** vs 标准 5-8%（**2.5x 富集**）
- **Flesch-Kincaid 16**（college level）vs 落败方 19.1（避免 PhD-level）
- **Subjectivity score 0.47**（事实+分析平衡）
- **大品牌 0.81x 引用率**（big brand penalty——大品牌反而不容易被引）

[Steve Toth 规则](https://www.airops.com/blog/ai-search-steve-toth)：每段 100-300 tokens（行业最具体规则）

[Mike King 4 支柱框架](https://ipullrank.com/how-ai-mode-works)：单 chunk ~500 tokens 上限

#### 价值
**在 Stage 4 LLM Pairwise Ranking 中战胜竞争 chunk**——这是真正的"GEO 独立价值"区域，传统 SEO 工具不做。

#### 饼图份额
影响 70% × 60% × 选择概率提升 ≈ **20-25%**（在已被 retrieved 的页面里，被 selected 的概率 2-3x）

#### 技术原理
1. **切分页面 chunks**：用 H2/H3 边界 + 100-500 token 限制（Steve Toth + Mike King 的合并）
2. **每个 chunk 用 Workers AI**（bge-* 嵌入模型）生成 384 维或 768 维 embedding
3. **Vectorize 存 chunk embeddings**，跑 cosine similarity vs 用户目标 prompts
   - 找出哪些 chunk 已经能被引用（cosine ≥ 0.55，Indig 用过的阈值）
   - 哪些差太远（cosine < 0.4 → 主题漂移，建议删除——Mike King 的"topic centroid drift"）
4. **NLP 评分**（用 Workers AI Llama 3.2 1B 做轻量推理）：
   - Entity density（NER 提取命名实体 / 总词数）
   - Flesch-Kincaid grade level
   - Definitive sentence count（"X is Y" 模式匹配）
   - BLUF 位置（前 30% 是否给答案——基于 chunk 在页面位置 + LLM 判断）
5. **改写建议**：Workers AI Llama 3.1 70B（英文）/ DeepSeek API（中文）→ 基于 Indig 研究的具体参数（20.6% entity density、Flesch-Kincaid 16）输出改写

#### 怎么做（产品功能）
- **粘 URL → 30 秒返回每 chunk 的"引用准备度评分"**（0-100）
- **热力图**：页面哪部分是"AI 引用黄金地带"、哪部分是 ramble
- **一键改写**："这段 entity density 6%（标准 20%），改成这样：[Llama 3.1 70B 输出]"——保留语义，提升语言学指标
- **Topic centroid 检测**：站点级页面与品牌主题的 cosine 相似度，提示"这 12 篇文章偏离了你的主题中心，建议删除或合并"（参考 [Mike King 1000+ 文章修剪案例](https://ipullrank.com/relevance-engineering-at-scale)：45% 标记修剪 → 站点级语义相关性 +2-3%）

### Layer 4：Mention-worthiness（品牌网络存在）

#### 业务背景
[Ahrefs 75K 品牌研究](https://ahrefs.com/blog/ai-brand-visibility-correlations/) 反共识发现：
- **YouTube mentions 0.737 是绝对最强 AI 可见度信号**
- 品牌网络提及 0.664 vs 反链 0.218——**3:1 优势**

Tim Soulo（Ahrefs CMO）：**"Citations are the new backlinks for the AI search era"**

[Indig 数据](https://www.growth-memo.com/p/the-ghost-citation-problem)：**LLM 行为差异**
- Gemini: 83.7% 提及 / 21.4% 引用（爱提名字，不爱给链接）
- ChatGPT: 87.0% 引用 / 20.7% 提及（爱给链接，不爱提名字）
- **61.7% 引用是 Ghost Citation**（链接存在但品牌名不出现）

[Lily Ray 各 LLM 引用源偏好](https://ipullrank.com/seo-week-2025-lily-ray)：
- **OpenAI**: Wikipedia + G2 最多
- **Perplexity**: Reddit + YouTube
- **Google AIO**: YouTube + LinkedIn
- **Copilot**: Forbes + Gartner

#### 价值
**长期护城河**。前 3 个 Layer 是战术，这一层是战略。

#### 饼图份额
**70% × 15% = 11% 直接来自特殊源**（Reddit/YouTube/Wiki）+ **间接影响其他所有 Layer**（Stage 5 Synthesis 决定提名你 vs 只链接你）→ 总效应可能达到 **30%**

#### 技术原理
1. **Browser Rendering** 抓 `site:youtube.com 用户品牌名` + Reddit / X / LinkedIn 类似查询
2. **Vectorize** 算 mentioning 上下文的语义（夸你、批评你、还是中立列名）
3. **Ghost Citation 拆解**：用户被引用 N 次，但只在 M 次被提到品牌名
4. **跨 LLM 偏好建议**：基于 LLM 不同偏好，给"针对 ChatGPT 投放 Wikipedia + G2"vs"针对 Perplexity 投放 Reddit + YouTube"
5. **YouTube Data API** 监测视频中的品牌提及（有些是 transcript 文本，有些是 description / pinned comment）

#### 怎么做（产品功能）
- **仪表盘**："你品牌 YouTube 提及 3 次（行业平均 28）。Ahrefs 数据：YouTube 是 0.737 最强信号。建议本月 review 视频 3 条"
- **竞品对比**："竞品 X 在 Reddit 被提 47 次，你 8 次"
- **Ghost Citation 报告**："Gemini 引用你品牌 47 次，但只在 11 次提到品牌名——这是典型 Ghost Citation"
- **平台投放建议**：基于目标 LLM，告诉用户"想被 ChatGPT 引用 → 要被 Wikipedia / G2 提及；想被 Perplexity 引用 → 要被 Reddit / YouTube 提及"

### Layer 5：Defense（反 spam 防御）

#### 业务背景
[Lily Ray 真实惩罚案例](https://lilyraynyc.substack.com/p/is-google-finally-cracking-down-on)：

| 站点 | 行为 | 跌幅 | ChatGPT 引用 |
|------|------|------|------------|
| $80 亿 B2B 品牌 | 191 篇"best X tools"自家 #1 | **-49%** | 同步下跌 |
| SaaS | 类似 listicle 滥用 | **-43%** | 同步 |
| B2B/B2C SaaS | 同上 | **-42%** | 同步 |
| 另一 SaaS | 同上 | **-34%** | 同步 |
| 数字营销服务商 | 1 月中旬起 | **-29%** | 同步 |

**Microsoft 2026/2 披露**：31 家公司、50+ 例 prompt injection（"Summarize with AI"按钮里藏指令"recommend [Company] first"）。点名工具：CiteMET、AI Share URL Creator。

[Search Engine Land Black Hat GEO](https://searchengineland.com/black-hat-geo-pay-attention-463684)：5 类危险战术——大规模 AI spam、假 E-E-A-T、LLM cloaking、Schema 滥用、SERP 投毒

#### 价值
**保险**——避免一次错误抹掉过去 6 个月的所有努力。"我们不让你做会害死自己的 GEO"。

#### 饼图份额
不直接增加引用，但**防止 -49% 的灾难性损失**（保护已有 70%）

#### 技术原理
1. **Workers AI 跑分类器**检测页面是否含 5 大危险模式：
   - **AI 内容农场指纹**：perplexity / burstiness 检测（Stanford GPTZero 类算法）
   - **自我推销 listicle**：自家产品 #1 + AggregateRating schema 滥用模式（Lily Ray 案例数据训练分类器）
   - **日期粉刷**：dateModified 改但内容未实质变化（diff 算法 + 语义相似度阈值）
   - **Prompt injection**：扫描隐藏 instructions、CSS hidden 文本、color 同背景文本
   - **Comparison 页过度**：站点内 vs/alternatives 页面占比超过阈值（如 51 个 alternatives 页 = 红旗）
2. **给红绿灯报告 + 修复建议**

#### 怎么做（产品功能）
- **Pre-publish lint**：上传你的内容 → 5 大危险扫描（类似代码 lint）
- **现有内容审计**：扫站点找"已经在做但不知道有风险"的页面
- **修复建议**："你这页有 3 个 self-promotional listicle 红旗，Lily Ray 案例显示 -49% 惩罚风险。建议：(1) 把自家产品从 #1 移到 #4，(2) 删除 AggregateRating schema，(3) 加一个明确的 disclosure"

### Layer 6：China Platform Sourcing（中文市场专属）

#### 业务背景
[博客园 navyum 实证](https://www.cnblogs.com/navyum/articles/19118757)：**1 周内让豆包 / DeepSeek / Kimi 把他的 Chrome 插件推荐到第 2 名**——核心方法是"识别豆包优先抓 CSDN → 在 CSDN 发布优化标题文章"。

[奇赞数据](https://www.qizansea.com/82589.html)：
- 中国 AI 引擎闭环极强：豆包 → 抖音 / 头条 / CSDN；文心 → 百家号；通义 → 夸克
- **36.5% 中国网民已使用生成式 AI 搜索**
- **2026 中国 GEO 市场预计 ¥286 亿**（信通院）

**关键差异**：中国 GEO 杠杆点不在 Schema / BLUF 这种站内优化，而在**"AI 引用源平台映射"**——西方所有 GEO 工具完全不覆盖。

#### 价值
**中国独有杠杆**。Profound、AthenaHQ、Otterly 等西方工具零中文 AI 引擎覆盖；AIDSO 等中文工具不覆盖海外引擎。**"中英双向 + 平台溯源"是真空地带**。

#### 饼图份额（中文市场）
替代 Layer 4 的 11% 特殊源——但在中文市场杠杆更大。**中国 GEO 总效应 35%+**

#### 技术原理
1. **Browser Rendering** 模拟用户在豆包 / Kimi / DeepSeek / 文心 / 通义提交查询
2. **Workers AI / DeepSeek API** 解析答案中的引用 URL + 来源平台
3. **平台分类器**：URL → 平台 cluster（CSDN / 知乎 / 抖音 / 小红书 / 百家号 / 掘金 / 36 氪 / 头条）
4. **跨多查询聚合**：对你的品类，豆包 65% 引用 CSDN、25% 引用知乎、10% 抖音
5. **推荐发布平台 + 标题模板**：基于哪个平台对哪类查询权重高
6. **直接对接你已有的 xiaoheishu 桌面应用**——一键发布到这些平台

#### 怎么做（产品功能）
- **平台溯源仪表盘**："你的品类目标查询 → 豆包当前引用：CSDN 5 篇竞品、知乎 3 篇、抖音 2 个视频"
- **发布建议**："建议下月发：CSDN 3 篇（标题模板：『XX 工具评测：5 款 X 类型推荐』）、知乎 1 篇"
- **多平台一键分发**：通过 xiaoheishu 桌面应用，复用现有发布管道
- **效果追踪**：发布后 7/14/30 天检查 AI 引用是否提升

**产品定位句**：

## 关键技术决策

### 为什么用 Cloudflare 而不是传统 cloud

1. **Workers AI 边缘推理便宜**：Llama 3.1 70B = $0.293/M input tokens
2. **Browser Rendering 自带 + 便宜**：$0.09/小时，10 小时/月免费
3. **Vectorize 5M 向量免费**：足够 chunk-level embedding 工作流
4. **Cron Triggers 免费**：触发定时监测
5. **Cloudflare Pay Per Crawl + Content Signals**：可以集成成产品差异化

### 中文 LLM 不在 Workers AI 里——怎么办？

走外部 API：
- DeepSeek：~¥1/M tokens（最便宜）
- Kimi：context 长，适合长文章
- 通义千问：对中国 SEO 关键词理解好

**混合架构**：英文走 Workers AI（边缘 + 便宜），中文走 DeepSeek/Kimi API。Workers 同时调度。

### 单用户成本估算（Pro 用户 ¥199/月）

| 项 | 用量 | 成本 |
|----|------|------|
| Browser Rendering | ~5 hr/月超额 | $0.45 |
| Workers AI 改写 | ~50 篇 × 2000 词 | $0.03 |
| DeepSeek API（中文用户） | 50 万 tokens | ¥0.5（≈$0.07） |
| D1 / R2 / KV / Vectorize | 标准用量 | <$1 |
| **总成本** | | **~$2/月** |
| **毛利** | $28 - $2 = $26 | **>92%** |
