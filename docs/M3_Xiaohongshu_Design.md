# M3 小红书自动发布系统设计

## 概述

基于 M1(Blog) + M2(微信) 的成功经验，设计 M3(小红书) 自动发布系统，实现：
- 用户提供内容 → AI优化 → 随机模板 → 自动配图 → 发布到小红书

## 核心功能对比

| 功能 | M1 Blog | M2 微信 | M3 小红书 |
|------|---------|---------|-----------|
| 内容优化 | ✅ | ✅ | ✅ |
| 随机模板 | ❌ | ✅ 8种 | ✅ 适配小红书风格 |
| 自动配图 | ✅ 封面生成 | ✅ 封面上传 | ✅ 多图组合 |
| 发布方式 | 自动部署 | 草稿 | 草稿/定时发布 |
| 图片处理 | 本地生成 | 微信CDN上传 | 小红书CDN上传 |

## 技术方案

### 方案选择

经过对开源仓库的研究，推荐基于 `xpzouying/xiaohongshu-mcp` (12.6k stars) 构建：

**选择理由**：
- 最活跃的开源小红书 MCP 项目
- 支持图文/视频发布
- 支持定时发布
- 支持公开/私密/粉丝可见
- 支持原创声明
- 有 Docker 部署方案

## 图片处理策略

### 小红书图片规范
- 比例：3:4（推荐）、1:1、4:3
- 尺寸：最小 600×800，最大 1280×1706
- 数量：1-9 张
- 格式：JPG、PNG

### 自动配图方案
1. **封面生成**：使用 AI 生成 3:4 比例封面
2. **多图组合**：内容分段 + 每段配一张图
3. **水印添加**：每张图加 🍄 Mushroom Blog 和 XStack18

## 内容优化策略

### 小红书风格特征
1. **标题**：
   - 前10字必须吸引眼球
   - 使用数字、emoji、悬念
   - 示例："🚨 这个AI工具让我效率提升300%！"

2. **正文**：
   - 短段落（每段不超过3行）
   - 多用 emoji 分隔
   - 重点内容加粗/高亮
   - 结尾引导互动

3. **标签**：
   - 3-5个精准标签
   - 热门标签 + 垂直标签
   - 示例：#AI工具 #效率提升 #打工人必备

## 发布流程

```bash
# 一键发布小红书
./publish-xhs.sh content.txt

# 流程：
# 1. 读取 content.txt
# 2. AI优化为小红书风格
# 3. 生成/选择配图
# 4. 随机选择模板
# 5. 上传到小红书CDN
# 6. 创建草稿
# 7. 返回预览链接
```

## 技术依赖

### 必需服务
```yaml
# docker-compose.yml
services:
  xiaohongshu-mcp:
    image: xpzouying/xiaohongshu-mcp:latest
    ports:
      - "3000:3000"
    volumes:
      - ./cookies:/app/cookies
      - ./logs:/app/logs
    environment:
      - LOG_LEVEL=info
```

### 首次登录
1. 启动 MCP 服务
2. 获取登录二维码
3. 小红书 App 扫码
4. Cookie 持久化保存

## 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 小红书封号 | 使用草稿模式，人工审核后发布 |
| MCP服务不稳定 | 本地 Docker 部署，不依赖第三方 |
| 图片上传失败 | 本地路径优先，失败转 HTTP |
| Cookie过期 | 自动提醒重新登录 |
