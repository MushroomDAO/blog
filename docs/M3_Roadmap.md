## Phase 8: 多图轮播支持（Future - 延后开发）

> **状态**: 待基础功能稳定后开发  
> **优先级**: 中  
> **依赖**: MCP 服务支持多图上传（已支持）

### 功能描述
支持一篇笔记包含多张图片（小红书支持 1-9 张）。

### 实现方案
```
文章 Markdown
    ↓
提取所有图片引用
    ↓
生成多张封面（或内页图）
    ↓
按顺序上传到 XHS CDN
    ↓
发布时传入图片数组
```

### 技术要点
1. **图片收集**: 从 Markdown 中提取 `![alt](path)` 引用
2. **图片处理**: 统一处理为 3:4 或 1:1 比例
3. **上传顺序**: 保持原文档中的图片顺序
4. **封面选择**: 第一张作为封面，其余为内容图

### API 支持
```python
# publisher.py 已支持多图
images = [
    "/path/to/cover.jpg",
    "/path/to/content1.jpg",
    "/path/to/content2.jpg"
]
publisher.publish(title, content, images, tags)
```

## Phase 9: 定时发布支持（Future - 延后开发）

> **状态**: 待基础功能稳定后开发  
> **优先级**: 低  
> **依赖**: MCP 服务支持定时发布（已支持 `schedule_at` 参数）

### 功能描述
预设发布时间，系统自动在指定时间发布。

### 实现方案
```
用户指定发布时间
    ↓
创建定时任务（cron/at）
    ↓
到达时间后自动执行发布
    ↓
发送通知（可选）
```

### 技术要点
1. **时间格式**: ISO8601（如 `2024-01-20T09:00:00+08:00`）
2. **调度器选择**:
   - macOS: `launchd` 或 `at`
   - Linux: `cron` 或 `systemd timers`
3. **持久化**: 使用 SQLite 存储待发布任务
4. **约束**: 小红书支持 1 小时至 14 天内的定时发布

### CLI 设计
```bash
# 定时发布
./publish-xhs.sh content.md --schedule "2024-01-20T09:00:00+08:00"

# 查看待发布队列
./publish-xhs.sh --list-scheduled

# 取消定时发布
./publish-xhs.sh --cancel <task-id>
```

## 关于"中文显示优化"的解释

**原意**: 确保小红书内容的中文显示效果最佳

**具体包括**:

1. **字体选择**
   - 封面图使用支持中文的字体（如思源黑体、苹方）
   - 避免使用默认英文字体渲染中文

2. **排版优化**
   - 中文段落不需要英文那么大的行间距
   - 适当调整字符间距（tracking）
   - 标点符号居中对齐

3. **字符限制**
   - 标题严格控制在 20 字以内（小红书限制）
   - 正文控制在 1000 字以内
   - 标签使用中文更符合用户习惯

4. **Emoji 使用**
   - 小红书用户喜欢 Emoji
   - 在标题和正文中适当添加
   - 避免过度使用显得廉价

**当前状态**: 基础优化已在 `optimizer.py` 中实现，字体优化待添加。
